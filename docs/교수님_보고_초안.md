**과제:** 8~12세 기능적 조음음운장애 아동을 위한 AI 음운변동 분석 기반 게임형 한국어 조음 교정 앱  
**문서 목적:** AI 엔진(음소 인식) 개발 방향·데이터·학습·진행 상황 보고  
**작성:** 장원준 (초안) · **일자:** 2026. 5.

## 1. 보고 요약 (Executive Summary)

본 프로젝트의 AI 핵심은 **단어·문장을 맞추는 일반 STT가 아니라**, 아동 발화를 **음소(IPA) 단위로 “들리는 대로” 인식**하고, **목표 발음과 비교**하여 조음 피드백을 제공하는 **「청자(Listener) 모델」**이다.

- **학습:** KsponSpeech(성인) + AIHub 어린이 음성(sn=266)의 **음성·전사**를 이용, 전사는 **g2p**로 **IPA 음소 라벨**로 변환 후 **Wav2Vec2-CTC** 파인튜닝
- **추론(목표):** 연습 단어(예: 「사과」)의 목표 IPA vs 아이 녹음의 예측 IPA → **정렬·PCC** → 게임형 피드백
- **범위:** 임상 진단 도구가 아닌 **발음 학습 보조** (임상 검증·LLM 자연어 피드백은 후속 단계)

---

## 2. 문제 정의 및 차별점

### 2.1 해결하려는 문제

- 학령기 아동은 조음 오류가 있어도 **스스로는 정확히 말했다고 인지**하는 경우가 많음.
- 치료실(주 1~2회) 외 **가정 연습**에서는 **음소 단위 객관적 피드백**이 부족함.
- 국내에는 **8~12세·한국어·아동 본인 사용·음소 단위** 조음 교정 앱이 부족함.

### 2.2 기존 STT/TTS와의 차별

| 구분 | 일반 STT | HOPE (본 엔진) |
|------|----------|----------------|
| 출력 | 한글 **단어·문장** | **IPA 음소** 시퀀스 |
| 목표 | “뭐라고 말했는지” | “**어떤 소리**로 말했는지” |
| 틀린 발음 | “사과”로 **고쳐 쓰기 쉬움** | “다과”면 **ㄷ 쪽 소리**로 남김 |
| 모델 | LM 강한 ASR 다수 | **Wav2Vec2-CTC (LM-light)** |

**차별점은 “음성을 쓴다”가 아니라**, **한국어 학령기 대상·음소 단위·목표와의 비교·게임 연동**이다.

## 3. 시스템 개요

```text
[모바일 녹음] → API → ① 음소 인식(Wav2Vec2-CTC)
                      → ② 정렬·PCC·음향 분석(OpenSMILE)
                      → ③ 게임·피드백(UI)
```

**본 학기 AI 집중 범위:** ① 학습 및 `final/` 체크포인트 산출 (GPU 학습: 팀원 이주석, RTX 4080)  
**팀 내 분工:** 데이터 파이프라인·학습 코드·문서(본인), 앱·게임(UI/기획), GPU 본학습(이주석)

## 4. 핵심 용어 (보고 시 참고)

| 용어 | 설명 |
|------|------|
| **음소** | 말의 가장 작은 소리 단위 (예: ㅅ, ㅏ, ㄱ) |
| **IPA** | 국제음성기호. 소리를 통일된 기호로 표기 (`s`, `a`, `k` …) |
| **전사** | 녹음 내용을 **한글 등 글자로 적어 둔 것** |
| **g2p** | Grapheme-to-Phoneme. **한글 전사 → IPA** 변환 (코드: `g2p_ko.py`) |
| **PCC** | Percent Consonants Correct. **자음정확도(%)** — 치료·연구에서 흔히 사용 |
| **정렬** | 목표 음소열과 예측 음소열을 **짝지어** 틀린 위치를 찾는 과정 |

**예시 (개념):**

- 목표 단어 「사과」→ g2p → `s a k w a` (대략)
- 아이가 「다과」처럼 발음 → 모델 출력 `t a k w a` (대략)
- 정렬 결과: 첫 음소 불일치 → “첫 소리를 ㅅ처럼” 피드백

## 5. 학습 데이터 및 라벨 생성

### 5.1 사용 데이터 (본학습 필수)

| ID | 출처 | 내용 | 역할 |
|----|------|------|------|
| **D1** | KsponSpeech (AI Hub 「한국어 음성」) | 성인 **음성 + 한글 전사** | 성인 음소 base |
| **D2** | AIHub 어린이 음성 **sn=266** (8~12세) | 아동 **음성 + 한글 전사** | 아동 음향 적응 |
| D3, D4 | 자유대화·한국어 아동 (옵션) | — | **본 Stage mix 미포함** |

※ D1·D2 모두 **「음성 파일 + 전사」** 세트이며, 음소 단위로 사람이 라벨링한 전용 DB는 사용하지 않음.

### 5.2 라벨 생성 파이프라인

```text
zip (incoming) → extract → manifest.jsonl (음성 경로 + transcript)
                              ↓
                    학습 시: g2p(transcript) → IPA 정답
                              ↓
                    Wav2Vec2-CTC 파인튜닝
```

- **정답 라벨:** 데이터셋에 IPA가 없으므로, **전사 → g2p → IPA 35토큰** (설계서 §5.2)
- **학습 가정:** 정상 발화 데이터이므로, 전사 ≈ 실제 발화. **미발음·조음장애 전용 학습 데이터는 사용하지 않음** (설계 통찰: 정상 학습 후 **비교**로 오류 검출).

### 5.3 Stage (설계서 §6.1)

| Stage | 데이터 mix | 목적 |
|-------|------------|------|
| 1-A | D1만 | 성인 base |
| 1-B | D1 60% + D2 40% | 아동 적응 |
| 2 | D1 50% + D2 50% | 최종 fine-tune |

**1차 목표:** Stage **1-B** 1회 본학습 (일정·GPU에 따라 1-A·2 추가)

## 6. 모델 및 학습 설정

| 항목 | 내용 |
|------|------|
| 사전학습 | `facebook/wav2vec2-xls-r-300m` (약 300M 파라미터) |
| 구조 | `Wav2Vec2ForCTC` (Hugging Face Transformers) |
| 토크나이저 | `Wav2Vec2CTCTokenizer` + 한국어 IPA vocab 35토큰 |
| 학습 | feature encoder **freeze**, CTC head·상위층 파인튜닝 |
| 권장 하이퍼파라미터 | lr 3e-5, batch 16×accum 2, bf16, max_steps Stage당 약 4만~5만 |
| 예상 GPU 시간 (4080) | Stage 1회 **약 3~4일** (설계서 기준, 데이터량·step에 따라 변동) |

**Whisper·텍스트 LLM을 음소 인식기로 사용하지 않는 이유:** 단어·문맥 보정으로 **틀린 발음이 “그럴듯한 단어”로 수렴**할 수 있어, **음소 단위 오류 검출 목적과 상충**하기 때문.

## 10. 교수님께 여쭙고 싶은 점 (선택 — 필요 시 수정)

**HOPE AI = 한국어 음소를 듣는 청자**
