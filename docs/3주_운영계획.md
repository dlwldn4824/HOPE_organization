# HOPE 3주 운영계획 (군복무 기간 대행안)

## 현재 모델 상태 (현실 버전)

- `stage1b-mix/final` 체크포인트는 로드/추론 정상 동작.
- 소규모 음성 테스트에서 `사과/다과`, `자두/차두`는 구분 신호 확인.
- 일부 케이스는 라벨 매핑/녹음 강도 영향이 커서 결과 변동.
- 결론: 완성품이 아니라 **데모 가능한 프로토타입 단계**.

한 줄 평가:

> 모델은 동작한다. 다만 실사용 수준 일반화 전이고, 녹음 실험 설계/데이터 품질 개선이 필요하다.

---

## 3주 핵심 목표

1. 평가 체계 안정화 (파일명-라벨 불일치 제거)
2. 녹음 품질 표준화 (16k mono, 한 파일 한 단어)
3. 재학습 1회 이상으로 성능 변화 확인
4. 데모 단어셋 확정 및 발표 자료 고정

---

## Week 1 — 데이터/평가 체계 고정

### 해야 할 일

- 녹음 규약 고정:
  - 단어쌍: `사과/다과`, `자두/차두`, `바다/파다`
  - 각 단어당 `ok 5개 + err 5개` (최소)
  - 파일명 규칙: `word_ok_01.wav`, `word_err_01.wav`
- 라벨 시트 작성:
  - 컬럼: `file`, `target_word`, `type(ok/err)`, `speaker`, `date`
- 평가 스크립트 실행:
  - 산출물: `docs/stage1b_mix_voice_eval.csv`, `docs/stage1b_mix_voice_eval_summary.json`

### 주간 산출물

- 단어쌍별 `ok/err` 평균 PER 표 1개

---

## Week 2 — 데이터 확장 + 재학습 1회

### 해야 할 일

- 화자 3명 이상으로 동일 규약 재녹음
- `stage1b` 재학습 1회 (step/배치 설정 유지)
- 재학습 모델로 동일 eval 재실행

### 주간 산출물

- 전주 대비 PER 변화표
- 개선/악화 단어 목록

---

## Week 3 — 데모 안정화

### 해야 할 일

- 데모용 단어쌍 3개 최종 확정
- 실패 케이스(혼동 단어) 재녹음 또는 제외 기준 확정
- 발표 자료 업데이트:
  - 현재 성능 수치
  - 한계/다음 단계

### 주간 산출물

- 교수님 보고용 1장 요약 업데이트
- 최종 데모 스크립트(발표자/대사/시연 순서)

---

## 역할 분담 (권장)

- 데이터 담당: 녹음 수집, 파일명/라벨 시트 관리
- 모델 담당: 추론/재학습 실행, CSV/JSON 생성
- 발표 담당: 결과 표/그래프 제작, 문서 업데이트
- 팀장: 주간 점검, 리스크/일정 관리

---

## 필수 운영 규칙

1. 파일명-라벨 불일치 금지
2. 한 파일 = 한 단어
3. 16kHz mono wav 통일
4. 주간 공유 지표 3개 고정:
   - 전체 PER
   - `ok` 평균 PER
   - `err` 평균 PER

---

## 팀 공지문 (복붙용)

우리 모델은 현재 "동작하는 프로토타입" 단계입니다.  
3주 목표는 성능 미세개선보다 **평가 체계 안정화 + 데모 신뢰도 확보**입니다.

- Week1: 녹음 규약/라벨 시트 고정 + baseline PER
- Week2: 화자 확대 + 재학습 1회 + 전주 대비 비교
- Week3: 데모 단어 확정 + 발표 자료 고정

필수 규칙: 파일명-라벨 일치, 16k mono, 한 파일 한 단어.  
매주 보고: 전체 PER / ok PER / err PER + 실패 케이스.
